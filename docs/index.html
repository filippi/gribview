Sample grib data
Filter, browse, plot, and extract GRIB data; export CSV time series and point values.
What gribview can do
  • Open GRIB files at full speed with OpenGL rendering and ecCodes decoding.
  • Reorganize, sort, append, merge, and resave GRIB archives.
  • Filter out unwanted messages to make GRIB archives more compact.
  • Select point values, track them over time, and export CSV time series.
  • Scrub levels/parameters, tweak colour maps, export PNGs, and inspect full metadata.
Quick install (macOS via Homebrew)
  • brew: brew tap filippi/gribview then brew install gribview.
Install options
  • macOS (brew): brew tap filippi/gribview then brew install gribview.
  • macOS (DMG): download the DMG above, drag Gribview.app to Applications. It is unsigned; you may need to allow it under System Settings → Privacy & Security → Open Anyway.
  • Windows: unzip the package above and run gribview.exe.
  • Linux (Debian/Ubuntu): download the .deb above and run sudo apt install ./gribview-1.1-Linux-x86_64.deb. This pulls libeccodes, libsdl2, libglew automatically. Other distros can use the tarball from the release and install those libs via their package manager.
How to run on macOS (unsigned app)
  1. Download the Gribview.dmg file using the button above.
  2. Double‑click the DMG in Finder and drag Gribview.app into your Applications folder.
  3. Try to launch gribview once from Applications (or Spotlight). macOS will warn that the app is from an unidentified developer and block it.
  4. Open System Settings → Privacy & Security. Scroll down until you see the message saying that “gribview was blocked from use”.
  5. Click Open Anyway, then confirm by clicking Open in the dialog that appears.
  6. From now on you can launch gribview normally, or just double‑click .grib/.grib2 files to open them in the app.
